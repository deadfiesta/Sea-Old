  • Understanding the 3 'E'

    🎯 User Goals

    Reviewing the experience as a productivity and collaborative platform

  • Effortlessness

    How easy was it to use?

    The need for SeaTalk to be as intuitive as possible is essential as our target audience are average tech consumers who may not be “app-savvy”. The last thing we want is to intimidate our users and drop our app at first sight.

  • Effectiveness

    How well of a job did it do?

    Features in SeaTalk should add value whenever possible. This means giving the users an end result better than their desired expectation of the action performed.

  • Efficiency

    How fast to get the job done?

    The nature of any productivity app like SeaTalk should assist users to accomplish more but with less work in less time. Every action should be seamless; simplified and autonomous, cutting down as many steps as possible.

  • 01

  • Staying organised with a clean and minimal interface

  • Clear and direct with the essentials upfront. Good contrast and accessibility while maintaining an organised interface.

  • 02

  • Slight color palette and style rework

  • Observing the design trends, modern designs are adopting saturated colors and simpler flat geometric shapes for a modern, clean and colorful aesthetic.

  • A guide of UI design trends for 2021 uxdesign.cc
  • 03

  • Preloading animation

  • Loading animation is a wonderful way to convey a brand concept, distract attention when waiting. Why not entertain the user while they wait?

  • Everything you need to know about loading animations medium.com
  • 04

  • A brief guided tour for onboarding

  • A simple optional guided tour on first launch can assist and orientate the user around the interface especially during the onboarding process. A simple guide to get the user started right away.

  • What is the best UX/UI pattern for your product tour? appecues.com

  • Chat List

  • Features that are located in the chat list panel

  • 05

  • The flow from searching messages to the chat

  • The auto focus and the little wiggle at the end is subtle but very helpful to bring attention to the specific chat bubble where the result was found.

  • Search interface: 20 things to consider uxplanet.org
  • 06

  • Modal windows for search result and adding new contacts/chat

  • Modals are a great way to avoid interrupting the user's workflow from the main window while drawing attention to the new information on screen. Users can multitask and easily return to their original task at hand.

  • 07

  • Collapsible Chat List

  • The option to collapse the chat list to free up screen real estate for more important windows. Responsive resizing should close the chat list past a break point.

  • 08

  • Visual indicator for status availability

  • When you send a message, it’s very useful to know whether or not it will be read right away. Presence status in chat gives you this capability. Managing user's expectations of responsiveness.

  • 09

  • Hover state triggers a "more-option" menu

  • Ease of access for option menu than having to do an extra right click. Raise awareness to users that more options are available at the same time.

  • 10

  • Conditional muting options

  • Giving users the options to be notified on different scenarios and conditions. i.e. push notifications on tagged only etc.

  • Chatroom

  • Features that are related or located in the chatroom window

  • 11

  • Quick reply menu on hover

  • Quickly reply to a specific text (especially helpful in group chats). No fuss no muss.

  • 06

  • Bookmarking messages

  • Let the user bookmark a message for reference in the future. Unlike “later” feature, it’s a saved message instead of a reminder for following up.

  • 12

  • Combined forward text sent as chat history object

  • Full details are embedded along with the messages. They’re easy to refer to and they don’t flood the chat room unnecessarily.

  • 13

  • Source of origin for single forwarded text

  • Providing context of the chat for the recipient. Sending forwarded text in the same fashion as any ordinary text can be confusing to the recipient.

  • 14

  • Reducing chat bubble max width

  • Reducing the horizontal travelling distance of the eyes when reading long messages when the app is expanded or in full view.

  • 15

  • Typing awareness indicator

  • Humanising communication and facilitating the linguistics of conversational turn-taking without impinging on each other. TDLR: Prevent interruption.

  • Why the typing awareness indicator is the worst bustle.com

  • Afterthought Features

  • Secondary features that may be helpful and go a long way

  • 16

  • Notification Inbox

  • Consolidated notifications regarding any attention required for the user specifically whenever the user was mentioned. This allows user to have an overview at a glance quickly.

  • 17

  • Send images as original file or compressed

  • Giving the user the option to send an image file in original file to retain quality as lossy compression can result in color degradation. This potentially can cause an issue for sharing color sensitive work.

  • 18

  • An @here mention for group chat

  • A type of mention that only notifies everyone except those who are offline and do not suppress an @here mention from a group chat.

  • 19

  • Edit chat and showing revision

  • Correct typos and revise text to look better especially when that message is pinned.
